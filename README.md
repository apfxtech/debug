# debug
Компонент `debug` — единая система логирования и журналирования, оптимизированная
под Arduino, C++ и Python. Цель компонента — предоставить единый
код и единый стиль логирования во всех проектах ApertureFox, чтобы упростить
отладку, мониторинг и сопровождение кода на разных платформах.

Ключевые возможности:
- Лёгкая интеграция в проекты на C++ и Arduino.
- Уровни логирования: ERROR, WARN, INFO, DEBUG, TRACE (настраиваются через `DEBUG_LEVEL`).
- Макросы для условной компиляции: в релизе неиспользуемые функции не будут собираться.
- Форматирование сообщений с временными метками и возможностью вывода HEX-дампов.

Цель проекта
- Единый код и единый стиль логирования во всех проектах ApertureFox.

Требования к стилю кода
1) Минимальный безотказный код:
	- Предпочтение простым, проверенным конструкциям.
2) Защита от «мусора» — компилируются только используемые функции:
	- Использовать макросы уровня `DEBUG_LEVEL` для условной компиляции.
	- Встроенные форматы/буферы должны быть компактными по памяти.
3) Поддержка трёх платформ:
	- Arduino (AVR/ESP и т.д.) — использовать `Serial` и `millis()` где нужно.
	- Desktop C++ (Linux/macOS/Windows) — использовать std::chrono, std::cout и файловые backend'ы.
	- Python (план) — лёгкие мнгопоточные биндинги в стиле logging.
